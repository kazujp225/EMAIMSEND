## 1. プロジェクト概要

### 1.1 コンセプト

#### コアバリュー
- **最速の第一印象**: 文章をペースト → CSS強化（SONNET） → プロ品質のHTMLメール即プレビュー＆即送信
- **適切な相手にだけ**: 150万件の法人リストを業種別に分類し、業界×役割にフィットする提案を無駄打ちなく届ける
- **信頼と配慮**: オプトイン、配信停止、ドメイン認証など到達率と法令順守を妥協しない

#### ポジショニング
従来の「複雑なMAツール」でも「到達率だけのメール会社」でもなく、"文章を貼って1通送る"の体験を最短で磨き上げ、そこから段階的にスケールできる"営業メール・ランチャー"

#### 主要ベネフィット（3本柱）
1. **早い**: 数十秒で見栄えの良い営業メールが完成
2. **当たる**: 業種別カテゴリで、的外れな送信を減らす
3. **安心**: 到達・退会導線・同意管理などの"事故らない仕組み"

### 1.2 目標指標（KGI/KPI）

#### KGI
- 無料体験（1通送信）の開始数: 月間1,000件（3ヶ月目標）
- 無料→有料転換率: 15%以上

#### KPI
1. ヒーローから「無料で1通送る」クリック率: 25%以上
2. 業種別デモ閲覧率: 40%以上
3. 体験導線クリック率: 30%以上
4. 送信完了率（フォーム完走）: 70%以上
5. ページ滞在時間: 平均2分以上
6. 直帰率: 50%以下

## 2. ブランディング

### 2.1 サービス名

**候補**:
- MailLaunch（メールローンチ）
- QuickSend（クイックセンド）
- InstaMail Pro（インスタメールプロ）
- フォーム営業モーヤメ（センドクラフト）
- OneClick Mail（ワンクリックメール）

**推奨**: フォーム営業モーヤメ

### 2.2 タグライン

**最終候補**:
「貼って、整えて、刺さる。営業メールを1クリックで。」

**代替案**:
- 「文章を貼るだけ。プロ級HTMLで、最適な相手へ。」
- 「最初の1通を最速で。業種別ターゲティングで無駄なく届く。」

### 2.3 ブランドボイス

#### トーン
- **プロフェッショナルだが親しみやすい**
- 技術用語は最小限に抑え、わかりやすい言葉で
- 押し付けがましくなく、ユーザーの判断を尊重
- 法令順守を強調しつつ、堅苦しくない

#### 文体ガイドライン
- 「です・ます」調
- 一文は50文字以内を目安
- 専門用語には必ず説明を添える
- 数値は具体的に（「すぐに」ではなく「30秒で」）

#### Q11: 到達率を高めるには？

```
【技術面】

【コンテンツ面】

【運用面】
• 配信停止率を5%以下に保つ
• バウンス率を3%以下に保つ
```

### 9.4 データモデル

#### users（ユーザー）
```sql
CREATE TABLE users (
  id VARCHAR(255) PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  plan VARCHAR(50) DEFAULT 'free', -- free / starter / scale / enterprise
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### emails（送信メール）
```sql
CREATE TABLE emails (
  id VARCHAR(255) PRIMARY KEY,
  user_id VARCHAR(255) REFERENCES users(id),
  to_email VARCHAR(255) NOT NULL,
  from_name VARCHAR(255) NOT NULL,
  from_email VARCHAR(255) NOT NULL,
  subject VARCHAR(500) NOT NULL,
  html_body TEXT NOT NULL,
  plain_body TEXT NOT NULL,
  status VARCHAR(50) DEFAULT 'pending', -- pending / sent / delivered / bounced / failed
  sent_at TIMESTAMP,
  delivered_at TIMESTAMP,
  opened_at TIMESTAMP,
  clicked_at TIMESTAMP,
  open_count INT DEFAULT 0,
  click_count INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user_id (user_id),
  INDEX idx_status (status),
  INDEX idx_sent_at (sent_at)
);
```

#### templates（テンプレート）
```sql
CREATE TABLE templates (
  id VARCHAR(255) PRIMARY KEY,
  user_id VARCHAR(255) REFERENCES users(id),
  name VARCHAR(255) NOT NULL,
  industry VARCHAR(50), -- manufacturing / it_saas / retail / etc.
  scene VARCHAR(50), -- new_business / follow_up / event / case_study
  subject VARCHAR(500) NOT NULL,
  body_text TEXT NOT NULL,
  is_public BOOLEAN DEFAULT FALSE, -- 公開テンプレートかどうか
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user_id (user_id),
  INDEX idx_industry (industry),
  INDEX idx_scene (scene)
);
```

#### contacts（連絡先リスト）
```sql
CREATE TABLE contacts (
  id VARCHAR(255) PRIMARY KEY,
  user_id VARCHAR(255) REFERENCES users(id),
  email VARCHAR(255) NOT NULL,
  company_name VARCHAR(255),
  contact_name VARCHAR(255),
  industry VARCHAR(50),
  is_unsubscribed BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, email),
  INDEX idx_user_id (user_id),
  INDEX idx_industry (industry),
  INDEX idx_unsubscribed (is_unsubscribed)
);
```

### 12.1 特定電子メール法対応

#### 必須対応項目
```
✓ 送信者情報の明記
  - 送信者名
  - 会社名
  - 住所
  - 電話番号
  - メールアドレス

✓ 事前同意（オプトイン）
  - 同意取得の記録
  - 同意日時の保存

✓ 配信停止手段の提供
  - 1クリックで配信停止
  - 配信停止の即時反映（自動）
  - 配信停止後の確認メッセージ

✓ 件名の適正表示
  - 誤認させる件名の禁止
  - 必要に応じて「広告」表示

✓ 表示義務
  - メール下部に送信者情報を自動挿入
  - 配信停止URLを自動挿入
```

```
✓ 利用目的の明示
  - プライバシーポリシーで明記
```
